# OBS HUD Templates

## 概要

OBS配信用HUDのテンプレートシステム。プロフェッショナルで洗練されたレイアウトを即座に適用できます。

## テンプレートカテゴリ

### 1. Minimal（ミニマル）
シンプルで洗練されたデザイン。必要最小限の情報を美しく配置。

- **Corner Stats**: 画面隅に統計情報を配置
- **Top Bar**: 上部中央にキーメトリクスを配置
- **Side Panel**: 縦型サイドパネルレイアウト

### 2. Detailed（詳細表示）
豊富な情報を見やすく配置。分析やレビューに最適。

- **Full Dashboard**: 包括的な統計情報ダッシュボード
- **Split View**: 統計とグラフを横並びで表示

### 3. Streaming（配信向け）
視聴者に見やすいデザイン。配信画面を邪魔せず情報を提供。

- **Bottom Ticker**: 画面下部にティッカー表示
- **Corner Overlay**: 控えめなコーナーオーバーレイ
- **Title Safe Area**: YouTubeやTwitchのサムネイル対応配置

### 4. Competitive（競技向け）
プロフェッショナルで見やすいデザイン。トーナメントや競技シーンに最適。

- **E-Sports HUD**: プロトーナメントスタイル
- **Scoreboard**: 見やすいスコアボードレイアウト
- **Analyst View**: 解説向けの詳細分析表示

## デザインコンセプト

### カラーシステム
- **勝利**: 緑系（#10b981） - エネルギーと成功を表現
- **敗北**: 赤系（#ef4444） - 注意と警告を表現
- **勝率**: 橙系（#f59e0b） - バランスと中立性
- **総試合数**: 青系（#06b6d4） - 信頼性と安定性
- **アクセント**: 紫系（#8b5cf6, #667eea） - プレミアム感と洗練性

### タイポグラフィ
- **フォントサイズ**: 16px〜32px（要素タイプに応じて調整）
- **フォントウェイト**: Bold（600-700）でコントラストを強調
- **行間**: 適切な余白で可読性を確保

### レイアウト原則
1. **視線の流れ**: Z型または F型パターンを意識
2. **グリッドシステム**: 8pxベースのグリッドで整列
3. **余白**: 適切なpadding（12px〜28px）で呼吸感を確保
4. **階層**: boxShadowとopacityで深度を表現

### 透明度・影
- **背景透明度**: 70%〜90% - OBSのゲーム画面を活かす
- **Box Shadow**: 0 4px 12px〜0 8px 24px - 浮遊感と存在感
- **Glow Effect**: カラフルな要素にはグロー効果で視認性向上

## 技術仕様

### 座標システム
- 1920×1080（フルHD）を基準
- 要素は絶対位置で配置
- スナップガイドで整列支援

### レスポンシブ対応
テンプレートはフルHD（1920×1080）を想定していますが、以下の考慮事項があります：
- OBSのブラウザソースでスケーリング可能
- 要素のscaleプロパティで個別調整可能
- paddingとfontSizeで視認性を調整

### パフォーマンス最適化
- GPU加速: `will-change`, `backface-visibility`
- 最小限のDOM要素
- 効率的な再レンダリング

## カスタマイズ

テンプレート適用後、以下をカスタマイズ可能：
- 位置（ドラッグ＆ドロップ）
- フォントサイズ
- 背景色・透明度
- パディング
- ボックスシャドウ
- 表示/非表示
- スケール（拡大率）

## ベストプラクティス

### 配信用途別推奨テンプレート
- **カジュアル配信**: Minimal系
- **本気配信**: Streaming系
- **大会配信**: Competitive系
- **解説配信**: Detailed系

### OBS設定推奨
1. ブラウザソースのサイズ: 1920×1080
2. カスタムCSS: 不要（テンプレートに含まれる）
3. FPS: 30以上推奨
4. ハードウェアアクセラレーション: 有効推奨
